【规范】编码规范（C++）

发表于 2018-03-05 | 分类于 cpp

1 命名约定

1.1 通用命名规则

  • 驼峰命名法：利用大小写更替来指示不同的单词，如：

    ImageFilter
    PixelType
    DataStorage
    NodePredicateProperty

  • 避免使用下划线，如：

    Image_Filer
    _Node

  • 变量名应表达代码背后的含义，如：

    BoundingBox::Pointer boundingBox = BoundingBox::New();

  • 拼写出全名而非缩写，如：

    mitk::DataNode* node;

  • 通用简称允许缩写，如：

    ROI表示 Region of Interest

1.2 类

  • 类名以大写字母开头

  • 类命名需依据以下一般性规则：

    class name =

如：

Accessor：类型之间的访问和转换，如NullScalarAccessor
Container：点或图像等对象的容器，如VectorContainer
Filter：参与数据处理管道的类，如AddImageFilter
Mapper：将数据从一种形式转换为另一种形式，如ContourMapper2D
Reader/Writer：读取/写入单个数据对象的类，如VtkSurfaceReader

  • 像ExampleClass这样的MITK类应该在命名空间mitk中，其相应的文件应该命名为mitkExampleClass.h/.cpp。

    mitk::DataStorage

  • Qt特定的MITK类（如QmitkListView）应在其类名中包含前缀Qmitk，并将其相应的文件命名为QmitkListView.h/.cpp。

    QmitkDataStorageComboBox

  • 头文件以.h结尾
  • 源文件以.cpp结尾
  • 模板类文件以.txx结尾

1.3 方法/函数

  • 以大写字母开头

  • 代码中引用类的方法时，应显式使用this->指针，如：

    mitk::DataStorage::SetOfObjects::ConstPointer all = this->GetAll();

1.4 信号/槽方法/函数

  • 槽函数命名依据如下一般性规则：

    On[发送信号的变量]信号类型;

如：

connect( loadImagePushButton, SIGNAL( clicked(bool ) ),SLOT( OnLoadImagePushButtonClicked( bool ) ) );
void mitk::Image::OnLoadImagePushButtonClicked( bool )
{
... Do something ...
}

  • 信号命名依据如下一般性规则：

    Signal方法名称;

如：

emit SignalFinishedSegmentation();

1.5 类数据成员

  • 类数据成员添加 m_ 前缀，且下划线后首字母大写，如：

    m_Volumes
    m_OffsetTable
    m_ImageMask

  • Qt类数据成员不加前缀，并以小写字母开头，如：

    loadImageButton
    closeImageAction

1.6 局部变量

  • 驼峰命名法，且第一个单词以小写字母开头，如：

    offset
    data
    slicesIt

1.7 全局变量

  • 应尽量减少全局变量的使用，如果必须，使用g_作为前缀，以便更好的区分局部变量。

1.8 常量

  • 在全局或类里的常量名称前加k，且除开头的k之外每个单词首字母均大写，如

    m_kImageMask
    g_kImageMask

1.9 Qt变量

  • GUI变量以Qt控件类型名结尾，如：

    QPushButton loadImagePushButton;
    QAction     closeImageAction;
    QCheckBox hideImageCheckBox;
    QRadioButton     binaryImageRadioButton;

1.10 类型命名

  • 类型名称的每个单词首字母均大写，不包含下划线，所有类型命名（class，struct，typedef，enum）均使用相同约定，如：

    MyExcitingClass
    MyExcitingEnum

  • Typedef声明的类型名以单词Type结尾，如：

    typedef TPixel PixelType;
    typedef itk::Image< TPixel, VImageDimension > ImageType;
    typedef std::listmitk::Image::Pointer ImageListType;

2 指针

2.1 指针的声明

  • 指针*/引用&应与变量连接，如：

    int *counter;
    int &counter;

2.2 智能指针 SmartPointer

  • SmartPointers必须用于以itk::Object作为基类的类

  • 将刚创建的实例分配给普通指针会导致崩溃，因为引用计数会立即减少到零，并且对象会被销毁

    itk::Object::Pointer object = itk::Object::New();

  • 当变量的作用域保留时，静态声明也被禁止并导致异常，因为当引用计数仍大于零时调用了析构函数
  • 请注意，使用智能指针需要在设置输入时使用真实（普通）指针。如果你想要返回一个新创建的智能指针（该指针不在类中）（例如，如果您编写了一个Clone方法），则必须返回一个智能指针（比较itkMacro.h）。如果智能指针保存在类中，则返回一个真正的（普通）指针就足够了。 -测试SmartPointer是否为空应使用IsNull（）和IsNotNull（）方法。简单的使用==NULL会发出警告。

3 命名空间

  • ARMD类应包含在命名空间armd内，如：

    #include “…”
    namespace armd
    {

    class TestClass
{
    ...
}

    }

4 代码布局和缩进

4.1 整体布局

  • 禁止使用 using namespace xx污染命名空间
  • 禁止使用尾随空格

  • 变量声明应该是每行一个声明，如

    int sliceNumber;
    char *stringName;
    ImageType::Pointer image;

  • 每一个限定符（public/protected/private）内，声明顺序如下：

顺序

内容

权限

1

typedefs和enums

public

2

常量

public/private

3

构造函数

public

4

析构函数

public

5

成员函数，含静态数据成员

public/protected/private

6

成员变量，含静态成员变量

public/protected/private

4.2 类布局

  • Copyright（.h，.cpp文件均需包含）

    // Copyright 2018 Ariemedi LLC. 版权
    // License(BSD/GPL/…) 许可证，BSD/GPL/MIT/…
    // Author: Li Wenjie 作者
    // This is … 文件内容简短说明

  • Include[A..Z]

    #include “… .h”

  • Namespace

    namespace armd
    {

  • Typedefs

    public:
    ….typedefs….

  • Methods

    public:
    ….methods….
    protected:
    ….methods….
    private:
    ….methods….

  • QT Signals

    signals:
    Signal…();

  • QT Slots

    public slots:
    On…();
    protected slots:
    On…();

  • 数据成员定义为私有，放在类结尾

    private/protected:
    ….class data members….
    };
    }
    #endif

5 大括号的使用

  • if，for，while，switch等必须使用大括号{}

  • 大括号单独占一行，如

    for ( unsigned int i = 0; i < 3; ++i )
    {
    … do something …
    }

  • 代码段只有一行时可以将其放在一行，如：

    if ( condition ) { foo = 1; }
    else if ( condition2 ) { foo = 3; }
    else { return; }

6 include保护符

  • #inlcude 保护符是一个特殊的构造，用于在处理#include指令时避免双重包含的问题
  • #inlcude 保护符的命名约定是：ClassName_h

7 其它

  • 头文件中尽量使用前置声明
  • 只将这些头文件包含在真正必要的头文件中，其余部分包含在实现文件中
