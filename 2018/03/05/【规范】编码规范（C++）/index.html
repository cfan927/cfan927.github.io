2.2 智能指针 SmartPointer

  • SmartPointers必须用于以itk::Object作为基类的类
  • 将刚创建的实例分配给普通指针会导致崩溃，因为引用计数会立即减少到零，并且对象会被销毁
  • 当变量的作用域保留时，静态声明也被禁止并导致异常，因为当引用计数仍大于零时调用了析构函数
  • 请注意，使用智能指针需要在设置输入时使用真实（普通）指针。如果你想要返回一个新创建的智能指针（该指针不在类中）（例如，如果您编写了一个Clone方法），则必须返回一个智能指针（比较itkMacro.h）。如果智能指针保存在类中，则返回一个真正的（普通）指针就足够了。 -测试SmartPointer是否为空应使用IsNull（）和IsNotNull（）方法。简单的使用==NULL会发出警告。

  • 禁止使用尾随空格

  • 每一个限定符（public/protected/private）内，声明顺序如下：
