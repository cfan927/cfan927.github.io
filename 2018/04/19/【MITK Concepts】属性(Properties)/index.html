【MITK Concepts】属性(Properties)

发表于 2018-04-19 | 分类于 mitk

编辑：赵志奇 审校：田兆瑞 原文链接：http://docs.mitk.org/nightly/PropertiesPage.html

属性

一个数据节点（data node）含有属性（properties），属性包含了MITK对节点处理的相关信息。属性提供了一个存储额外信息的空间。所谓的额外信息，它不是实际数据的一部分，因此没有理由包含在数据或文件当中。但是我们可能会在渲染（例如：传递函数）或交互（例如：节点名称）的时候用到它们。 属性可以通过以下方式读取和设置：

/*1: 读取属性*/
mitk::Property::Pointer readProperty = node->GetProperty("color");
//从数据节点中读取属性“颜色”

mitk::ColorProperty::Pointer colorProperty = dynamic_cast<mitk::ColorProperty*>(readProperty);
//将这个属性强制转化为子类型ColorProperty

/*2: 写入属性*/
mitk::BoolProperty::Pointer newProperty = mitk::BoolProperty::New( true );
//创建一个新的属性（这里的例子创建的是一个新的布尔型的属性）

node->SetProperty( "IsTensorVolume", newProperty );
//将这个属性添加到这个属性到含有关键字“IsTensorVolume”的节点

1.属性服务（Property Services）

属性服务能够扩展MITK基本属性系统的能力，目前有四种属性服务，它们是别名（aliases）、说明（descriptions）、扩展（extensions）、过滤器（filters）。下面将会对它们做一个简单的叙述。关于它们更详细的信息可以在相应的API文档中找到，所有的服务都有完整的文档说明。 所有的属性服务都是通过C++ Micro Services的功能提供的。因此，您可以使用所有模块和插件中的属性服务。

1.1 属性别名(Property Aliases)

你可以将一个属性别名添加到某个由属性视图（Property View）显示的属性名称上，而不是添加到真实的属性名称上。属性名称可以有多个别名，在这种情况下，属性会在属性视图中显示多次。另外，别名很容易被限制在某些数据节点类型中。 属性别名由服务接口mitk::IPropertyAliases提供。

1.2 属性说明(Property Descriptions)

如果属性有相关说明，属性视图将会在底部显示当前选定属性的说明。说明被解析并显示为富文本格式（Rich Text）。 属性说明由服务接口mitk::IPropertyDescriptions提供。

1.3 属性扩张(Property Extensions)

属性扩展是将元数据（Meta data）附加到属性的一种非常通用的方法。它们的唯一要求就是继承自mitk::PropertyExtension。属性视图通过搜索已知的属性扩展去设置属性的编辑器，比如mitk::IntPropertyExtension和mitk::FloatPropertyExtension，可以指定数值属性的有效范围。 属性扩展由服务接口mitk::IPropertyExtensions提供。

1.4 属性过滤器(Property Filters)

属性过滤器是一个相当强大的工具，它通过属性视图去过滤显示的属性。有一个单一的全局过滤器，并可以在其上为特定的数据节点类型添加过滤器。过滤器由黑名单条目和白名单条目组成，即黑名单条目被过滤掉，白名单条目是过滤后剩下的唯一属性。如果两个列表都包含条目，则只显示白名单条目，也就是未列入黑名单的条目。 属性过滤器由服务接口mitk::IPropertyFilters提供。

2.模块独立属性列表

本节根据属性使用的地方列出MITK中大部分的已知属性。并不是每个节点都需要全部属性（甚至不需要太多属性）。

2.1 文件管理

  • path：文件加载的物理路径（physical path）
  • name：数据管理器中的节点名称
  • selected：数据管理器是否选择该节点

2.2 通用渲染属性

color：对表面（surface）、灰度图像、任何应该被渲染的内容着色（一般默认是白色）。有一个特殊的mitk::ColorProperty，你可以使用Getter/Setter方法去使用它。颜色由三个值（RGB）定义，范围介于0.0和1.0之间。

备注：

在一个映射器中，你可以使用下面的代码来获取颜色：

float rgb[3]={1.0f, 1.0f, 1.0f};
GetColor( rgb, BaseRenderer );

注意：

如果你设置了属性“查找表”（Lookup Table）,该属性不会影响图像。在这种情况下，将会使用用户定 义的查找表。

通过几何确定平面上的重采样范围：

  • 重采样网格对应当前的世界几何（world geometry）。这意味着输出的2D图像的间隔（spacing）取决于当前选定的世界几何，而不是图像自身。
  • 重采样网格对应当前的输入几何。这意味着无论当前选择的世界几何是什么，输出的2D图像的间隔都直接从关联的输入图像中导出。
  • layer：控制哪个图像在另一个图像的上面。如果两张图像占据同样的空间，较高层的图像将会挡住较低层的图像。到目前为止，它适用于图像和点集。层属性仅适用于相似的数据类型。点集总是渲染在图像的前面，所以，层属性不会对其造成任何影响。
  • levelwindow：一般情况下，灰度图像的值在0到255之间。因此，默认的窗口边界被设置为0和255。为了显示一定灰度范围内的图像，比如50-200，该属性就被用来调整最小和最大边界的值。
  • LookupTable：该属性包含用户定义的查找表，用来将标量值映射到颜色值。举个例子：如果一幅图像包含值128，在生成的图像中，查找表将此值映射为红色（255，0，0）。

注意：

如果设置了“LookupTable”属性，“color”属性不会再影响图像。 MITK中设置查找表的例子： ```cpp #include #include #include #include […] vtkSmartPointer vtkLUT = vtkSmartPointer::New(); vtkLUT->SetRange(100,200); //define your table here vtkLUT->Build(); //pass the table to MITK mitk::LookupTable::Pointer mitkLookupTable = mitk::LookupTable::New(); mitkLookupTable->SetVtkLookupTable(vtkLUT); mitk::LookupTableProperty::Pointer LookupTableProp = mitk::LookupTableProperty::New( mitkLookupTable ); result->SetProperty( “LookupTable”, LookupTableProp ); result->Update(); 


 - opacity：节点、图像、表面等的透明度值（Alpha值）。取值范围在0.0和1.0之间。 #### 备注： 在一个映射器中，你可以使用以下代码来获取不透明度： ```cpp float opacity=1.0f; GetOpacity( opacity, BaseRenderer );

  • reslice interpolation：该属性仅在旋转模式（swivel model）或十字准线旋转（crosshair rotation）中生效。插值模式“Nearest”、“Linear”、“Cubic”都是可用的，并且沿着旋转平面影响像素结果。
  • texture interpolation：该属性切换为纹理插值。如果启用，图像像素之间的边缘会模糊；如果禁用，边缘保持清晰。
  • visible：触发节点、图像、表面的渲染的开关。

2.3 表面渲染属性

  • back color：在2D中，表面外部法向量的颜色
  • back normal lenth (px)：在2D中，以像素为单位的法向量的长度（当减小它，使用front color代表的颜色会更短？）
  • color mode：（来自VTK）控制标量数据如何映射到颜色。默认情况下（ColorModeTo-Default）,无符号字符标量被视为颜色，不通过查找表进行映射，其它的都要经过映射。设置ColorModeToMapScalars意味着所有标量数据都要经过查找表映射。
  • draw normal 2d：在2D中，决定是否显示法向量
  • front color：在2D中，表面内部法向量的颜色
  • front normal lenth (px)：在2D中，以像素为单位的法向量的长度（当减小它，使用back color代表的颜色会更短？）
  • invert normals：在2D中，转换front/back 法向量
  • line width：在2D中，控制表面与平面相交的线的宽度和法线的宽度
  • material.ambientCoefficient：3D中的环境光系数
  • material.diffuseCoefficient：3D中的漫反射光系数
  • material.interpolation：选择用于表面构建的插值算法
  • material.representation：选择绘制mesh的表现形式（表面Surface、线框Wireframe、点云Point Cloud）
  • material.specularCoefficient：增加/减少非漫反射
  • material.specularPower：控制非漫反射的百分比
  • material.wireframeLineWidth：线框表示中线的宽度
  • scalar mode：（来自VTK）控制过滤器如何使用标量的点数据和单元属性数据。默认情况下（ScalarModeToDefault），过滤器将使用点数据，如果没有点数据可用，则使用单元数据。或者，您可以显式设置过滤器以使用点数据（ScalarModeToUsePointData）或单元数据（ScalarModeToUseCellData）。 您还可以选择从点字段数据（ScalarModeToUsePointFieldData）或单元字段数据（ScalarModeToUseCellFieldData）中的数组中获取标量数据。如果标量来自字段数据数组，则必须在调用GetColors之前调用SelectColorArray。将ScalarMode设置为使用“字段数据”（ScalarModeToFieldData）时，必须调用SelectColorArray来选择字段数据数组用于对单元着色。在这种模式下，如果多边形数据具有三角形条带（triangle strips），那么字段数据将被视为由条带中三角形形成的每个微型单元的单元数据，而不是整个条带的。
  • scalar visibility：（来自VTK）打开/关闭标志以控制是否使用标量数据为对象着色
  • selected：是否选择该节点
  • shader：该着色器用于表面渲染，当前的选项是“fixed”和“mitkShaderLightning”。

2.4 体渲染属性

  • TransferFunction：包含用于着色图像的传递函数
  • volumerendering：是否要进行体渲染
  • volumerendering configuration：选择Composite还是MIP
  • volumerendering.cpu.ambient：环境光
  • volumerendering.cpu.diffuse：增加/减少光的散射
  • volumerendering.cpu.specular：增加/减少非漫反射

  • volumerendering.cpu.specular.power：控制非漫反射的百分比

  • volumerendering.gpu.amibent：使用gpu，同cpu

  • volumerendering.gpu.diffuse

  • volumerendering.gpu.specular

  • volumerendering.gpu.specular.power

  • volumerendering.ray.ambient：使用ray,同cpu

  • volumerendering.ray.diffuse

  • volumerendering.ray.specular

  • volumerendering.ray.specular.power

  • volumerendering.gpu.reducesliceartifacts：减少切片伪影

  • volumerendering.gpu.usetexturecompression：使用纹理压缩

  • volumerendering.usegpu：是否使用gpu进行渲染

  • volumerendering.uselod：是否使用细节层次机制（Level Of Detail）

  • volumerendering.usemip：是否使用最大亮度投影（Maximum Intensity Projection）
  • volumerendering.useray：是否使用光线投射（raycasting）

备注：

Uselod可以用usegpu，usemip，useray来激活，但后者中的任何一个都不能与其中的另一个一起使用。

2.5 点集属性

  • close contour：是否连接轮廓（点集中的点与点之间的连接段connecting pieces）的第一个点和最后一个点
  • contourcolor：决定轮廓（点集中点与点之间的连接段）的颜色，仅在“show contour”处于激活状态时才可见。
  • contoursize：表示轮廓的直径（点集中点与点之间的一种管道），仅在“show contour”处于激活状态时才可见。
  • distance decimal digits：设置点与点之间的欧氏距离的小数点位数，可以通过激活“show distances”显示距离。
  • point 2D size：2D视图中的点的位置由交叉十字表示。“point 2D size”决定了这个十字的尺寸大小。
  • point line width：2D视图中的点的位置由交叉十字表示。“point line width”决定了十字的宽度。
  • pointsize：3D视图中的点的位置由球体表示。“pointsize”决定了这球的直径（大小）。
  • selectedcolor：设置点集中选定点的颜色
  • show angles：如果“显示轮廓”处于激活状态，则可以显示两个轮廓部分之间的角度。
  • show contour：通过在点集中的点之间绘制连接段来连接这些点。
  • show distance lines：显示轮廓的所有角度和线条（2D视图），即使它们不在当前切片视图上。
  • show distances：在下列点之间绘制线条（2D视图），并显示这些点之间的欧氏距离。
  • show points：点在视图中是否可见
  • updateDataOnRender：如果设置为“true”，则在渲染之前更新点集。如果点集是过滤器管道的一部分，这也会导致对管道的更新，这可能是不希望得到的，所以可以将其设置为false。

2.6 几何2D属性

  • draw edges：决定是否应在2D平面的边缘绘制管道（tubes）。默认为true。

2.7 平面图属性

平面图使用多个属性来创建更加友好的用户操作：

  • planarfigure.default.helperline.color：在该图既不被选中，鼠标也不悬停在上面的情况下，控制辅助线的颜色。
  • planarfigure.default.helperline.opacity：在该图既不被选中，鼠标也不悬停在上面的情况下，控制辅助线的不透明度。
  • planarfigure.default.line.color：在该图既不被选中，鼠标也不悬停在上面的情况下，控制线条的颜色。
  • planarfigure.default.line.opacity：在该图既不被选中，鼠标也不悬停在上面的情况下，控制线条的不透明度。

  • planarfigure.default.markerline.opacity：在该图既不被选中，鼠标也不悬停在上面的情况下，控制标记线的不透明度。
  • planarfigure.default.outline.color：在该图既不被选中，鼠标也不悬停在上面的情况下，控制轮廓线的颜色。
  • planarfigure.default.outline.opacity：在该图既不被选中，鼠标也不悬停在上面的情况下，控制轮廓线的不透明度。
  • planarfigure.hover.helperline.color：在鼠标悬停在图上但没有选中的情况下，控制辅助线的颜色。
  • planarfigure.hover.helperline.opacity：在鼠标悬停在图上但没有选中的情况下，控制辅助线的不透明度。
  • planarfigure.hover.line.color：在鼠标悬停在图上但没有选中的情况下，控制线条的颜色。
  • planarfigure.hover.line.opacity：在鼠标悬停在图上但没有选中的情况下，控制线条的不透明度。
  • planarfigure.hover.marker.color：在鼠标悬停在图上但没有选中的情况下，控制标记的颜色。
  • planarfigure.hover.marker.opacity：在鼠标悬停在图上但没有选中的情况下，控制标记的不透明度。
  • planarfigure.hover.markerline.color：在鼠标悬停在图上但没有选中的情况下，控制标记线的颜色。
  • planarfigure.hover.markerline.opacity：在鼠标悬停在图上但没有选中的情况下，控制标记线的不透明度。
  • planarfigure.hover.outline.color：在鼠标悬停在图上但没有选中的情况下，控制轮廓线的颜色。

  • planarfigure.hover.outline.opacity：在鼠标悬停在图上但没有选中的情况下，控制轮廓线的不透明度。

  • planarfigure.selected.helperline.color：在图被选中的情况下，控制辅助线的颜色。

  • planarfigure.hover.helperline.opacity：在图被选中的情况下，控制辅助线的不透明度。

  • planarfigure.selected.line.color：在图被选中的情况下，控制线条的颜色。
  • planarfigure.hover.line.opacity：在图被选中的情况下，控制线条的不透明度。
  • planarfigure.selected.marker.color：在图被选中的情况下，控制标记的颜色。
  • planarfigure.hover.marker.opacity：在图被选中的情况下，控制标记的不透明度。
  • planarfigure.selected.markerline.color：在图被选中的情况下，控制标记线的颜色。

2.7 分割属性

下面这些属性控制如何渲染分割结果：

  • binary：控制图像是否被视为二值/分割图像
  • binaryimage.hoveringannotationcolor：如果分割结果有一个属性表明它的悬停设置为true，则控制标注的颜色
  • binaryimage.hoveringcolor：如果分割结果有一个属性表明它的悬停设置为true，则控制分割结果的颜色
  • binaryimage.selectedannotationcolor：如果分割结果有一个属性表明它被选定设置为true，则控制标注的颜色
  • binaryimage.selectedcolor：如果分割结果有一个属性表明它被选定设置为true，则控制分割结果的颜色
  • outline binary：控制分割结果是绘制为轮廓（on）还是实体（off）
  • outline width：如果绘制的是轮廓，控制轮廓的线宽

  • showVolume：控制分割结果的体数据是否在2D视图中显示为标注

    分割视图使用的分割结果来检查图像是否应该被视为分割结果。
