【MITK从入门到放弃】Windows下编译MITK

发表于 2018-04-08 | 分类于 mitk

1. 版本要求

  • System：Win10
  • IDE：MSVC 2015 Community
  • MITK：2016.11
  • CMake：3.7.2
  • Qt：5.7.1

2. 安装工具

安装VS 2015、Cmake、Qt等，常规流程，版本正确即可（vs2015 64bit binary安装版本，无需自己编译），这里不再赘述

3. 开始编译MITK

a. 下载mitk windows版源代码（下载地址http://mitk.org/wiki/Downloads）

解压后得到的源码目录如下：

b. 打开cmake-gui，设置源码路径及生成路径

c. 点击Condigure按钮，配置文件

第一次点击Configure按钮，会弹出提示框选择编译器，这里需要选择跟mitk版本一致的编译器（比如我们用的mitk是64位版本，那么这里就要选择Win64编译器），一定注意不要选错 点击Finish后等待，出现Configure done表示Configure成功

d. 点击Generate按钮，生成配置文件/makefile，输出Generate done后则表示Generate成功

e. 点击Open project打开VS项目

f. 对ALL_BUILD右键选择“生成”

g. 等待编译完成即可

4. 常见问题

a. CTK build error

该问题多半是由于CMake版本太高导致 CTK源代码中有一处多了一个结尾大括号}，高版本CMake（如3.9.x）会报错，而3.7.x会忽略这个语法错误

b. ITK build error

该问题是由于build目录长度过大，ITK目录长度要求小于50个字符

c. 警告被视为错误

中英文编码错误 解决方法：双击错误提示，打开错误文件，选择“文件”-“高级保存选项”，选择“Unicode-代码页1200”，保存文件，再次对ALL_BUILD右键“生成”即可

d.VS2015的安装错误

MITK编译失败也可能是由于VS2015缺少相应的插件造成的。 解决方法：注意要勾选编程语言中的Visual C++选项和windows和web开发选项。

e.要提前把第三方库放到”…/MITK_bin/MITK_build/ep/src”目录下，不然VS在编译MITK时会自动下载第三方库，使得编译时间特别长。

5. 打开MITK

进入如下目录： 打开startMitkWorkbench_debug.bat/startMitkWorkbench_release.bat即可打开MITK 注意：.bat是一个批处理文件，对其“右键”以文本方式打开，截取部分内容如下： 第一行设置临时环境变量，这些路径都是MITK运行是需要的链接库，第二行为需要打开的可执行文件
